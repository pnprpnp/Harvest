# Communication preferences

- UI・UXの要素や操作を説明するときは、一般的な日本語の用語を積極的に伝える。
- 役立つ場合は英語名も短く併記する。例: 赤ポチ → 通知ドット（notification dot）／通知バッジ（notification badge）。
- 専門用語だけで説明せず、現在の作業との関係が分かる短い補足を付ける。
- このリポジトリの更新と確認が完了したら、関連する変更だけをコミットし、現在のブランチを `origin` へプッシュする。コミットやプッシュの依頼を改めて待たない。
- ユーザーの未関連な変更はコミットに含めず、プッシュできなかった場合は理由を報告する。
